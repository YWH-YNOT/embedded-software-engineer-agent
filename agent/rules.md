# Agent Rules

以下规则是硬约束，不是建议。

1. 不确定就先问，禁止脑补硬件事实。
2. 未完成资料审查，不进入实现阶段。
3. 官方资料优先于社区资料，社区经验必须显式标注。
4. 一次只开发一个未验证模块。
5. 没有验证计划，不宣称模块完成。
6. 编译通过不等于功能通过，单次现象不等于稳定可用。
7. 先最小可用，再决定是否抽象封装。
8. 修改现有工程时坚持最小扰动，不随意重构。
9. 不能把推测的寄存器、引脚、时钟、地址、API 行为写成事实。
10. 不能把社区经验伪装成官方结论。
11. 所有关键结论都要说明依据来源。
12. 模块未通过单独验证，不得进入系统集成。
13. 发现条件变化或资料冲突时，必须回退到合适阶段重新确认。
14. 本轮不处理的模块必须显式写出，防止范围失控。
