# 关键追问机制

本文件定义 Agent 在资料不完整时应该怎么问。

目标不是“问得多”，而是“问得关键”。问题数量尽量少，但必须覆盖会改变开发路径的阻塞项。

## 提问原则

- 先问会影响架构、驱动写法、时序、中断、DMA、验证方式的问题
- 先问能阻止错误实现的问题，再问优化问题
- 同类问题尽量合并，避免零碎追问
- 用户已明确给出的信息，不重复追问
- 问题必须带目的，避免“为了显得认真”而发问

## 优先级定义

| 优先级 | 含义 | 处理要求 |
| --- | --- | --- |
| P0 | 阻塞项 | 不解决就不能进入当前模块开发 |
| P1 | 高优先级约束 | 会显著影响驱动写法、资源分配或验证路径 |
| P2 | 优化项 | 不阻塞最小可用实现，但会影响健壮性、性能或后续扩展 |

## 通用追问框架

优先按以下顺序检查是否需要发问：

1. 平台身份：芯片准确型号、封装、板卡版本、资料版本
2. 硬件约束：供电、电平、时钟源、复位、启动方式、调试接口
3. 引脚与实例：外设实例号、引脚复用、片选、地址、总线连接
4. 软件基础：SDK/HAL 版本、现有工程、构建工具、RTOS 情况
5. 驱动路径：轮询 / 中断 / DMA、缓存策略、超时策略、错误恢复
6. 验证条件：测试设备、观测手段、成功判据、失败排查入口

## 建议提问格式

推荐把问题压缩成一组，按优先级输出：

```text
P0:
1. ...
2. ...

P1:
1. ...

P2:
1. ...
```

## 模块追问模板

### UART

| 优先级 | 追问 |
| --- | --- |
| P0 | 使用哪个 UART / USART 实例？TX / RX 引脚与复用配置是否已确认？ |
| P0 | 波特率、数据位、校验位、停止位、流控要求是什么？ |
| P1 | 采用轮询、中断还是 DMA？日志口和业务口是否共用？ |
| P1 | 是否有收发超时、帧格式、换行或协议边界要求？ |
| P2 | 是否需要低功耗唤醒、空闲中断收包、环形缓冲区？ |

### SPI

| 优先级 | 追问 |
| --- | --- |
| P0 | 使用哪个 SPI 实例？SCK / MOSI / MISO / CS 引脚是否确认？ |
| P0 | SPI 模式是 CPOL/CPHA 哪一组？字长、位序、最大时钟是多少？ |
| P1 | CS 由硬件还是软件控制？是否有多从设备共享总线？ |
| P1 | 采用轮询、中断还是 DMA？是否要求全双工或突发传输？ |
| P2 | 是否需要总线互斥、事务封装、错误恢复或超时处理？ |

### I2C

| 优先级 | 追问 |
| --- | --- |
| P0 | 使用哪个 I2C 实例？SCL / SDA 引脚、上拉、电平是否确认？ |
| P0 | 目标器件 7-bit 地址是多少？速率是 Standard / Fast / Fast Mode Plus？ |
| P1 | 是否需要 repeated start、clock stretching、寄存器地址宽度或分页写入？ |
| P1 | 总线上是否挂了多个设备？是否需要总线恢复策略？ |
| P2 | 是否要用 DMA；是否存在休眠唤醒、低功耗或热插拔约束？ |

### ADC

| 优先级 | 追问 |
| --- | --- |
| P0 | 采集的是哪个通道，输入范围、参考电压、分压或前端电路是什么？ |
| P0 | 目标分辨率、采样速率、单次/连续模式要求是什么？ |
| P1 | 是否需要校准、过采样、DMA 搬运或定时器触发？ |
| P1 | 数据用于阈值判断还是精确测量？允许误差是多少？ |
| P2 | 是否需要多通道扫描、滤波、均值或异常点处理？ |

### DMA

| 优先级 | 追问 |
| --- | --- |
| P0 | 当前外设对应哪个 DMA 控制器 / 通道 / 流 / request？官方资料是否已确认？ |
| P0 | 传输方向、数据宽度、缓冲区长度、normal/circular 模式是什么？ |
| P1 | 完成中断、半传输中断、错误中断是否需要？ |
| P1 | 是否有 cache、对齐、内存区域、双缓冲等限制？ |
| P2 | 是否需要链式 DMA、scatter-gather 或性能优化？ |

### PSRAM

| 优先级 | 追问 |
| --- | --- |
| P0 | PSRAM 的准确型号是什么？数据手册是否已提供？ |
| P0 | 接口类型是 SPI / QSPI / OPI 哪一种？工作电压、最大时钟、时序模式是什么？ |
| P1 | 初始化流程、读写命令、burst 模式、dummy cycles、等待状态要求是什么？ |
| P1 | 是否作为缓存区、帧缓冲还是通用堆；容量边界与地址映射如何处理？ |
| P2 | 是否需要 DMA 直连、缓存一致性处理、读写性能测试和老化测试？ |

### 中断

| 优先级 | 追问 |
| --- | --- |
| P0 | 中断源是什么，触发条件是上升沿/下降沿/双边沿还是电平？ |
| P0 | 中断优先级是否已有系统级约束？ |
| P1 | ISR 内只置位标志还是要完成数据搬运？是否会和 DMA / RTOS 交互？ |
| P1 | 是否存在共享中断、去抖、误触发、清中断时序要求？ |
| P2 | 是否需要统计丢中断、延迟或抖动数据？ |

### RTOS

| 优先级 | 追问 |
| --- | --- |
| P0 | 使用哪种 RTOS，版本是什么，当前模块运行在哪个任务上下文？ |
| P0 | 中断到任务的同步方式是什么，是否允许在 ISR 中调用相关 API？ |
| P1 | 任务优先级、栈大小、堆限制、tick 频率是否已知？ |
| P1 | 当前模块是否需要互斥锁、消息队列、事件组或零拷贝缓冲区？ |
| P2 | 是否有实时性、低功耗或看门狗喂狗要求？ |

## 终止追问的条件

当满足以下条件时，应停止追问并进入下一阶段：

- 当前模块的 P0 问题已解决
- P1 问题已足够支撑最小可用实现
- 剩余 P2 问题不会阻塞当前模块验证

如果 P0 问题无法解决，应明确暂停当前模块，而不是强行继续。
