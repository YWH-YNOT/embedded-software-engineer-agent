# Discussions Seed Topics

本文件提供可直接复制到 GitHub Discussions 的首批话题草稿。

## 1. Q&A

建议标题：

`Q&A：首次使用 embedded-software-engineer-agent 时你卡在哪一步？`

建议首帖：

本话题用于收集首次使用仓库时的实际问题。提问时请尽量附上：

- 芯片 / 平台
- 当前阶段
- 是否有原理图
- 是否有官方手册
- 你已经使用了哪些模板或示例

如果问题涉及明确缺陷，请改开 Issue。

## 2. Roadmap

建议标题：

`Roadmap 讨论：下一阶段最应该优先补什么？`

建议首帖：

本话题用于讨论仓库下一阶段应优先补齐哪些能力。请尽量把建议落到以下类别之一：

- 新示例
- 新 skill
- 模板增强
- 协作基建
- 规则收敛

请同时说明收益、影响范围和维护成本。

## 3. Platform Requests

建议标题：

`Platform Requests：你最希望优先支持哪个 MCU / SoC 平台？`

建议首帖：

如果你希望仓库优先补充某个平台，请在回复中提供：

- 平台名称
- 官方参考手册 / 数据手册
- 官方 SDK / HAL / BSP
- 你希望覆盖的模块
- 你手里是否有真实硬件

没有官方资料支撑的平台请求通常不会优先进入仓库。

## 4. Design Reviews

建议标题：

`Design Reviews：对 AGENTS.md、skills、templates 的结构性反馈`

建议首帖：

本话题适合讨论结构性设计问题，例如：

- `AGENTS.md` 和 `agent/` 的边界是否清晰
- 某个 skill 是否触发过宽或过窄
- 模板字段是否过多或过少
- 示例是否覆盖了关键阶段

请尽量引用具体文件和具体段落。

## 5. Show And Tell

建议标题：

`Show and Tell：你如何把这个仓库接入自己的嵌入式项目`

建议首帖：

如果你已经在自己的项目中使用了这个仓库，欢迎分享：

- 你的平台和项目类型
- 你实际使用了哪些文件
- 你删掉了什么、保留了什么
- 哪些规则最有帮助
- 哪些地方还不够

重点分享真实工作流，而不是只贴一段结果截图。
