# How To Ask

仓库欢迎问题，但问题越具体，越容易得到可执行的回复。

## 提问前先准备

至少说明以下内容：

- 芯片 / 平台准确型号
- 当前阶段：资料审查 / 关键追问 / 模块拆解 / 单模块开发 / 单模块验证 / 集成
- 当前模块是什么
- 是否有原理图
- 是否有官方参考手册 / 数据手册
- 是否有官方 SDK / HAL / 例程
- 是否有现有工程或最小可复现代码
- 你预期 Agent 做什么
- 你实际看到的问题是什么

## 不推荐的提问方式

- “帮我直接做完整工程”
- “STM32 项目怎么写”
- “SPI 有问题，帮看一下”
- “给我一个万能 Prompt”

这些问题的问题在于：范围太大、资料太少、无法验证。

## 推荐的提问方式

- “STM32F103C8T6，UART1 日志口，PA9/PA10，已有原理图和 HAL。请按仓库规则先做资料审查，再给单模块验证清单。”
- “STM32F411 + TMP102，I2C1 已能寻址但温度读数异常。请按单模块调试清单帮助我定位。”

## 用 Issue 还是 Discussions

- 明确缺陷、规则冲突、模板问题：用 Issue
- 使用疑问、路线讨论、设计取舍：用 Discussions
- 涉及敏感信息或凭据：不要公开发，参见 [`SECURITY.md`](../SECURITY.md)
