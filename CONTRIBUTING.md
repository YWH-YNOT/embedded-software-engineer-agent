# Contributing

感谢你为 `embedded-software-engineer-agent` 提交改进。

本仓库的目标不是堆叠文档数量，而是维护一套可执行、可审查、可复用的嵌入式 Agent 规则。因此，贡献内容必须优先服务以下原则：

- 官方资料优先
- 阶段控制明确
- 一次只推进一个未验证模块
- 每一步都有完成标准
- 验证闭环优先于“生成更多内容”

## 适合提交的内容

- 修正规则中的歧义、冲突或不可执行表达
- 补充新的模块追问模板、验证模板或集成模板
- 增加新的平台示例，但要明确资料前提与边界
- 改进 README、Issue 模板、PR 模板等协作基础设施

## 不建议提交的内容

- 营销化文案
- 大量重复文档
- 脱离官方资料约束的“万能提示词”
- 超出 V1 范围却没有明确边界说明的内容
- 为了“看起来完整”而增加的空模板

## 文档风格要求

- 中文为主，必要处保留准确的英文术语
- 句子短，结论明确，可直接执行
- 规则要能被检查，尽量避免模糊表达
- 模板要能直接填写，不能只是目录装饰
- 不把社区经验写成官方事实

## 提交流程

1. 先查看 README 与现有 `agent/`、`templates/`、`examples/` 是否已覆盖你的内容。
2. 优先在现有文件内增量修改，避免新建同类重复文件。
3. 如果改动会改变 Agent 行为，请同步更新示例或模板。
4. 提交 PR 时，明确写清楚：
   - 解决了什么问题
   - 为什么现有规则不足
   - 修改后的行为边界
   - 如何验证该修改没有破坏主工作流

## PR 最低要求

- README 中的目录结构与实际文件一致
- 新增模板必须可填写
- 新增规则必须能映射到实际工作阶段
- 示例内容必须与 `agent/system.md` 和 `agent/output_contract.md` 一致
- 不引入与当前目标无关的复杂脚本或框架

## 对新平台支持的要求

如果你要提交新的 MCU / SoC 平台示例，请至少提供：

- 芯片或平台型号
- 官方参考手册 / 数据手册
- 官方 SDK / HAL / BSP 或官方例程来源
- 基本硬件连接信息
- 该平台在 V1 范围内的建议支持边界

## 沟通建议

- 讨论规则时，优先给出具体工程场景
- 讨论模板时，优先展示实际填写结果
- 讨论示例时，优先指出它验证了哪条规则
