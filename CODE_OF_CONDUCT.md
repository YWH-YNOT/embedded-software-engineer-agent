# Code Of Conduct

## 目标

本仓库的讨论应服务于一个明确目标：维护一套受控、可审查、可验证的嵌入式 Agent 模板，而不是制造噪声。

## 我们希望的协作方式

- 讨论具体工程问题，而不是抽象口号
- 质疑规则时给出可复现的场景和资料前提
- 指出缺陷时同时说明影响范围和预期行为
- 尊重事实、证据和边界，不拿猜测当结论
- 优先帮助后来者快速理解仓库怎么用、边界在哪

## 不接受的行为

- 人身攻击、侮辱、威胁、骚扰
- 故意发布误导性技术信息
- 在已知资料不足时强行推动“直接出完整工程”
- 用营销化、夸大化语言替代技术论证
- 刻意刷屏、灌水或重复提交低质量内容

## 争议处理原则

- 先回到具体文件、具体规则、具体示例
- 先看是否违反 `AGENTS.md`、`agent/`、`templates/` 的既有约束
- 对流程问题优先开 Discussion，对明确缺陷优先开 Issue
- 如果争议无法在公开讨论中收敛，由维护者按仓库目标做最终裁决

## 适用范围

本准则适用于：

- Issues
- Pull Requests
- Discussions
- 示例、规则、模板相关的仓库协作

## 执行

维护者有权对明显违反本准则的内容执行编辑、隐藏、锁定讨论或拒绝合并。

如果你认为某个互动已经偏离仓库目标，请联系维护者，并附上相关链接和上下文。
