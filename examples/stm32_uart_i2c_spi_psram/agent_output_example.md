## 5. 开发或调试建议
- 先不要写任何寄存器配置或 HAL 初始化代码，因为 I2C 器件和 PSRAM 型号都未确认。
- 当前最关键的两份资料是温度传感器数据手册和 PSRAM 数据手册。
- UART 如果只是日志口，可以先按最小可用路径规划为 TX-only 或 TX/RX 基础收发；如果有协议需求，再决定是否引入中断或 DMA。
- PSRAM 模块应拆成两层：`SPI2 传输层` 和 `PSRAM 设备层`，不要直接混成一个驱动。

## 5. 开发或调试建议
- 建议顺序：`UART1 日志口 -> I2C1 传输层与 TMP102 读数 -> SPI2 传输层 -> PSRAM 设备层 -> 系统集成`。
- UART1 先做日志口，是因为它本身简单且能为后续模块提供调试输出。
- PSRAM 不应越过 SPI2 传输层直接实现，否则后续定位问题时无法区分是总线问题还是设备协议问题。
- 在每个模块通过前，不要把下一个模块标记为“完成”。

## 4. 当前只处理的模块
- 模块名称：SPI2 传输层 + PSRAM 设备层
- 模块边界：SPI2 负责基础读写事务；PSRAM 负责设备初始化和基础地址读写
- 依赖条件：PSRAM 数据手册已确认，UART1 可输出日志
- 本轮不处理：DMA 优化、缓存一致性优化、大块吞吐优化

## 5. 开发或调试建议
- 先把 SPI2 事务层单独验证，通过逻辑分析仪确认时钟、片选和数据方向。
- 再根据 PSRAM 数据手册实现设备层初始化与基础读写序列，不要跳过官方时序要求。
- 初版验证先做固定地址写入 / 读回 / 多地址 walking pattern，不要一开始就做大容量压力测试。
- 如果读写不一致，先分离为三类问题：SPI 模式错误、命令序列错误、片选/时序错误。
