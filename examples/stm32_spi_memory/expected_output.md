# 期望输出示例：STM32 SPI 存储器

## 1. 当前阶段
- 阶段：Phase 3 -> Phase 4
- 当前目标：先完成 SPI2 事务层验证，再完成 PSRAM 基础读写验证

## 2. 已确认信息
| 类别 | 内容 | 依据 |
| --- | --- | --- |
| 平台 | STM32F407VET6 | 用户输入 |
| 外设实例 | SPI2 | 原理图 + 引脚表 |
| 引脚 | PB13 / PB14 / PB15，PB12 为 CS | 原理图 + 引脚表 |
| 设备资料 | PSRAM 型号和官方数据手册已提供 | 用户输入 |
| 当前策略 | 先轮询，不用 DMA | 用户输入 |

## 3. 缺失信息
| 优先级 | 缺失项 | 为什么重要 | 需要用户提供什么 |
| --- | --- | --- | --- |
| P1 | 最小验证容量 | 影响测试样本大小和稳定性检查范围 | 给出至少一组读写容量目标 |

## 4. 当前只处理的模块
- 模块名称：SPI2 传输层 + PSRAM 设备层
- 模块边界：SPI2 负责事务层，PSRAM 负责初始化和基础读写
- 依赖条件：器件手册、引脚和片选关系已确认
- 本轮不处理：DMA、缓存一致性、性能调优

## 5. 开发或调试建议
- 先抓 SPI 波形，确认模式、片选和方向。
- 再根据 PSRAM 数据手册实现最小命令序列。
- 先做固定地址读写和连续地址读写，不直接做高吞吐优化。

## 6. 验证清单
| 测试项 | 方法 | 成功判据 | 失败排查路径 |
| --- | --- | --- | --- |
| SPI 事务层 | 逻辑分析仪抓 SCK / MOSI / MISO / CS | 波形与预期模式一致 | 查 SPI 模式、GPIO 复用、CS 控制 |
| 固定地址读写 | 写入测试模式后读回 | 读回数据与写入一致 | 查命令序列、时序、dummy cycles |
| 连续地址读写 | 连续地址写入递增数据后读回 | 无错位、无丢字节 | 查地址处理、事务边界、CS 时序 |

## 7. 本轮产出与边界
- 本轮产出：SPI 存储器最小 bring-up 路径
- 明确不产出：DMA 版 SPI、系统级缓存管理

## 8. 是否允许进入下一阶段
- 结论：有条件允许
- 原因：只有读写闭环通过后，模块才算完成
- 进入下一阶段前必须满足：记录至少一组固定地址读写与一组连续地址读写结果
